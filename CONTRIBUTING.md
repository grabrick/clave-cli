# Contributing to Clave

Спасибо за интерес! Ниже — что нужно для разработки.

## Окружение

- Rust + `cargo` (stable).
- `claude` и `codex` CLI — установлены и залогинены (нужны для реальных прогонов;
  для тестов можно подменить моками через `CLAVE_CLAUDE` / `CLAVE_CODEX`).
- Python 3 + `pyte` (`pip3 install pyte`) — для проверки inline-рендера.

## Сборка и проверки

```bash
cargo build --release        # бинарник target/release/clave
cargo test                   # юнит-тесты
cargo fmt                    # формат (без конфигов, стандартный)
cargo clippy                 # линт
```

Дешёвая проверка логики оркестрации без траты токенов:

```bash
./spec-clave --dry-run "<задача>"     # печатает план вызовов CLI, агентов не дёргает
```

Проверка живого inline-рендера (панели, отступы, скрытие футера) через эмулятор
терминала `pyte` — он работает, потому что ручной рендер (`src/render.rs`) не шлёт
CPR-запрос курсора:

```bash
python3 scripts/render_check.py target/release/clave <CLAVE_HOME>
```

(`CLAVE_HOME` — временный каталог с `config` и `chats/`; см. шапку скрипта.)

## Архитектура (коротко)

Два слоя: bash-движок `spec-clave` (двухагентный цикл планирования) и Rust TUI
(`src/`). Главное:

- **Рабочий язык интерфейса — русский.** Каждая видимая строка идёт через
  `lang.choose("ru", "en")` — голых строк в UI быть не должно.
- `model/` — чистые типы, `app/` — состояние/поведение, `ui/` — только рендер
  (читает `&App`, не мутирует), `worker.rs`/`storage.rs`/`auth.rs` — функции.
- Без async-рантайма: потоки + `mpsc`-каналы, главный цикл поллит события.
- Новая slash-команда = 3 места (`CommandSpec` в `COMMANDS`, ветка в
  `handle_command`, токен в `command_has_handler`), иначе падает тест палитры.

## Перед PR

- `cargo fmt && cargo clippy && cargo test` — зелёные.
- Если меняешь рендер/панели — прогони `scripts/render_check.py`.
- Сообщения коммитов — по-английски, в повелительном наклонении (как в истории).
